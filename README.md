## 为什么是 V2.0

`V2.0` 不是对旧单页工具做一次零散修补，而是把文献筛选流程重新整理成更贴近真实科研场景的工作台：

- 更清楚的入口结构：首页、双人复核登录页、主工作台拆分明确
- 更稳妥的去重策略：硬重复自动移除，疑似重复进入人工复核
- 更贴合研究流程：支持单人筛选与正式双人复核两种路径
- 更适合真实项目：保留 `30,000+` 记录规模处理能力，支持规则重跑与结果导出

当前在线默认入口已经切换到 `V2.0`，旧 `v1.7` 页面仅作为历史版本保留。

## 这个工具解决什么问题

| 常见问题 | V2.0 的处理方式 |
|------|------|
| 文献量一大，浏览器卡顿或无法持续处理 | 继续使用 IndexedDB + Web Worker + 虚拟列表的大规模处理架构 |
| 自动去重不透明，担心误删文献 | 改为双层去重：`硬重复自动移除` + `疑似重复人工复核` |
| 双人复核时状态容易冲突、互相覆盖 | 主审 / 副审路径与共享项目状态已补稳 |
| 全文复核里摘要显示不完整，英文内容还要手工翻译 | `V2.0` 已补强跨行摘要解析，并在全文复核弹窗新增“翻译本条文献” |
| 上传成功后页面不显示、步骤衔接断掉 | `V2.0` 已修复上传展示、步骤切换、页面滚动和显示链路问题 |
| 中英文入口混乱，英文页面跳错 | 英文首页、双人模式入口、登录页可见性和路由已修复 |
| 最终要交 PRISMA 图和结果明细 | 支持 PRISMA 2020 SVG 导出和结果数据导出 |

## V2.0 核心亮点

### 1. 更像真实科研项目，而不是单页 demo

- 首页、登录页、工作台分离，默认路径更清楚
- 单人模式与双人复核模式分流明确
- 适合从“导入文献”一路走到“导出结果”的完整流程

### 2. 去重逻辑更稳，也更容易解释

`V2.0` 将去重从页面脚本中拆出为独立的 [`dedup-engine.js`](./dedup-engine.js)，并采用双层输出：

```text
第一层：硬重复
基于更严格的证据规则自动移除，保持保守自动删除策略

第二层：疑似重复
不直接删除，而是显式抛给人工复核
```

这意味着 `V2.0` 的重点不是“尽可能多自动删”，而是更符合科研场景里的审慎原则。

### 3. 大规模能力保留，流程可用性更强

- 继续支持 `30,000+` 文献量级
- 上传后结果展示与页面滚动更稳定
- 示例导入、真实文献上传、步骤推进链路已补齐
- 双人复核下的共享项目初始化和退出清理已修复

### 4. 双语入口和正式使用路径更清晰

- GitHub Pages 根路径默认进入 `V2.0`
- 英文首页进入双人模式时会进入英文登录页
- 英文页面的中英混显与隐藏显示问题已修复

## 核心能力

### 多格式文献导入

支持 `CSV / TSV / RIS / ENW / BibTeX / RDF / TXT / NBIB`，可混合上传，多来源文献可在同一工作流中处理。

### 双层去重

- 硬重复自动识别并移除
- 疑似重复进入人工复核队列
- 去重结果更适合科研中的“保守自动化”原则

### 规则化筛选

- 语言筛选
- 年份范围
- 关键词包含 / 排除
- 标题 / 作者 / 期刊条件
- 规则重跑与结果复核衔接

### 单人筛选与双人复核

- 支持单人独立筛选流程
- 支持主审 / 副审双人复核
- 共享项目状态更稳定，减少协作返工
- 全文复核弹窗支持查看完整摘要与“翻译本条文献”

### PRISMA 2020 导出

- 支持 PRISMA 2020 流程图导出
- 支持研究结果明细导出
- 适合后续报告整理与投稿材料准备

## 标准工作流

```text
Step 1  导入文献
        支持多文件、多格式混传，自动跨库去重

Step 2  配置筛选规则
        语言 / 年份 / 关键词 / 期刊，保存为 YAML 可复用

Step 3  自动筛选结果
        查看通过文献，支持“修改规则重跑”

Step 4  人工审核
        快捷键 1-6 选择排除原因，进度自动保存

Step 5  导出
        PRISMA 2020 流程图（SVG） + Excel 详细报告
```

## 基准与客观变化

以下数据来自仓库内基准报告 [`docs/benchmarks/dedup/post-implementation-benchmark-report.md`](./docs/benchmarks/dedup/post-implementation-benchmark-report.md)。

| 指标 | v1.7 | V2.0 | 说明 |
|------|------|------|------|
| 自动删除精确率 | `1.000` | `1.000` | 保持零误删的保守自动删除策略 |
| 综合 duplicate-like recall | `0.583` | `0.917` | 对真实重复 / 近重复发现能力明显提升 |
| 综合 Candidate F1 | `0.737` | `0.957` | 疑似重复候选质量更高 |
| 真实 RDF hard recall | `0.667` | `1.000` | 在真实数据上补回更多应识别重复 |
| 真实 RDF candidate pairs | `0` | `1` | 能把该人工复核的情况显式抛出 |

### 如何理解“性能”和“效率”

- 目前没有单独证据表明 `V2.0` 在纯导入吞吐上显著快于 `v1.7`
- 本轮提升主要在正确性、稳定性、可解释性和真实流程效率
- 换句话说，这一版更大的进步是“更适合真正拿来做研究”，而不是单纯追求跑得更快

## 技术架构

```text
┌─────────────────────────────────┐
│  index.html / workspace.html    │  ← UI 层（步骤向导 + 虚拟滚动）
└────────────────┬────────────────┘
                 │
        ┌────────▼────────┐
        │     app.js      │  ← 业务逻辑层（规则引擎 + 流程控制）
        └────┬───────┬────┘
             │       │
     ┌───────▼──┐ ┌──▼──────────┐
     │ db-worker│ │parser-worker│  ← Web Worker 层（后台线程）
     └───────┬──┘ └──┬──────────┘
             │       │
        ┌────▼───────▼────┐
        │   IndexedDB     │  ← 数据层（30,000+ 篇本地存储）
        └─────────────────┘
             ↑
        dedup-engine.js    ← V2.0 独立去重引擎（双层输出）
```

### 核心模块说明

| 模块 | 作用 |
|------|------|
| `dedup-engine.js` | `V2.0` 独立去重引擎，负责硬重复与疑似重复输出 |
| `db-worker.js` | IndexedDB CRUD、批量插入 / 分页查询 / 事务管理 |
| `parser-worker.js` | 8 种格式解析、DOI / 标题归一化、流式处理大文件 |
| `virtual-list.js` | 虚拟滚动组件，只渲染可见区域 100 条 |
| `app.js` | 主流程控制、筛选规则、导出和状态管理 |
