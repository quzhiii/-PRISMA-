# PRISMA 文献筛选工具下一轮 UI / 产品重构 PRD

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将当前产品从“功能丰富的工具原型”升级为“看起来可信、专业、值得使用的研究工作台”，提升首次理解率、首次使用率、工具信任感与商业转化可能性。

**Architecture:** 默认入口保持为 `literature-screening-v2.0/index.html`，双人协作入口为 `literature-screening-v2.0/login.html`，核心工作台为 `literature-screening-v2.0/workspace.html`，共享设计系统集中在 `literature-screening-v2.0/style.css`。本轮不改变核心筛选逻辑，只重构表达层、信息层级和页面任务流。

**Tech Stack:** Static HTML, CSS, vanilla JavaScript, existing sessionStorage/localStorage language and workflow logic.

---

## 1. 背景

当前版本已经具备较完整的功能链路，但存在以下问题：

- 首页无法快速建立专业信任感
- 工具页信息层级偏乱
- UI 不能支撑高价值科研工具定位
- 功能虽然存在，但用户难以快速理解“为什么值得使用”
- 双人协作与商业卖点没有被充分表达

本轮重构不以“纯美化”为目标，而以提升以下结果为目标：
- 首次理解率
- 首次使用率
- 工具信任感
- 商业转化可能性

## 2. 产品目标

### 2.1 核心目标
将当前产品从“功能丰富的工具原型”升级为“看起来可信、专业、值得使用的研究工作台”。

### 2.2 本轮不做
- 不新增复杂业务功能
- 不重写核心筛选逻辑
- 不改动 PRISMA 导出核心机制
- 不进行大规模技术架构迁移

## 3. 用户问题

### 用户当前感知问题
1. 一眼看不出产品的核心价值
2. 页面虽然有很多信息，但不知道从哪里开始
3. 双人协作价值不够突出
4. 页面缺少“专业科研工具”的信任感
5. 首页、登录页、工具页之间缺少统一的产品叙事

## 4. 本轮重构范围

- 首页 `index.html`
- 登录页 `login.html`
- 工具页 `workspace.html`
- 共享样式系统 `style.css`

## 5. 重构原则

1. 首屏必须结果导向，而不是功能堆砌
2. 每个页面只保留一个主任务
3. 一级信息服务于转化，二级信息服务于理解
4. 视觉层级必须体现专业性、稳定感、可信度
5. 所有页面必须统一中英文体验
6. 首页语言状态必须自然传递到后续页面

## 6. 页面级需求

### 6.1 首页重构需求

#### 目标
让用户在 5 秒内明白：
- 这是什么产品
- 适合谁
- 有什么结果
- 下一步该做什么

#### 必须保留
- 品牌标识
- 语言切换
- 开始使用入口
- 双人协作入口
- 核心卖点展示
- 工作流展示

#### 必须调整
1. 首屏文案改为结果导向
2. CTA 更明确
3. 首页卖点顺序按用户价值排序，而不是技术实现排序
4. 增加“适合谁”区块
5. 增加“可信度”区块（本地离线、PRISMA、双人复核）

#### 建议结构
- Header
- Hero
- Core value strip
- Who it's for
- Core features
- Workflow
- Trust section
- Footer

### 6.2 登录页重构需求

#### 目标
把“双人协作”从普通功能入口，升级成有专业价值的模式入口。

#### 必须保留
- 主审 / 副审角色
- 项目 ID 流程
- 原有会话逻辑
- 项目状态检测

#### 必须调整
1. 更清晰地解释主审和副审的区别
2. 更清晰地解释项目 ID 的意义
3. 更突出“双人协作”的价值而不是只是登录
4. 全部英文/中文状态必须一致
5. 按钮文案、placeholder、提示信息全部语言同步

#### 登录页要传达的价值
- 双人独立审查
- 分歧识别
- Kappa 一致性分析
- 更适合正式系统综述流程

### 6.3 工具页重构需求

#### 目标
让用户进入工具页后立刻进入任务，而不是先看一堆说明。

#### 必须保留
- 五步流程
- 所有核心 ID / JS hooks
- 所有关键操作按钮
- 原有业务逻辑

#### 必须调整
1. 去除不必要的顶部介绍型大块内容
2. Step 1 首屏只突出“开始”
3. 每个 Step 标题下方只保留一句“为什么现在要做这一步”
4. 信息面板减少装饰，强调任务优先级
5. 结果页强调“结果解释”，而不是只是数字

#### Step 级方向
- Step 1：让上传和示例数据更直观
- Step 2：让规则配置更少认知负担
- Step 3：让统计结果更像“结论”
- Step 4：让协作与人工复核更像正式审查流程
- Step 5：让导出结果更像“可交付产出”

## 7. 转化率视角的文案要求

### 首页
不要写“支持多格式导入”，要写“减少前期整理时间，直接进入筛选流程”。

不要写“支持 Kappa”，要写“让双人复核更可追踪，并量化一致性”。

不要写“本地离线”，要写“研究数据留在本地，更适合敏感项目和课题组内部使用”。

## 8. 信任感模块要求

本轮必须新增或强化以下内容：

1. 隐私说明
2. 适用场景说明
3. 输出结果示意
4. 双人协作价值解释
5. PRISMA 标准说明

## 9. 体验目标

### 用户体验成功标准
- 用户第一次进入首页能立即理解产品定位
- 用户可以不读长文案就知道下一步操作
- 用户切换语言后，后续页面保持一致
- 双人模式英文/中文都完整可用
- 工具页不会让用户觉得“混乱”或“像原型”

## 10. 验收标准

### 功能验收
- 首页为默认入口
- 首页语言状态能传递到登录页和工具页
- 登录页英文状态完整显示
- Step 1 到 Step 5 不受 UI 调整影响
- 所有关键 ID / onclick / scripts 保持可用

### 产品验收
- 首页首屏可清晰表达产品价值
- 登录页能解释双人协作模式
- 工具页顶部无多余口号型说明块
- 页面层级更清晰，重点更集中
- 全站中英体验一致

## 11. 优先级

### P0
- 首页首屏重构
- 登录页价值表达重构
- 工具页顶部与 Step 1 重构
- 语言继承与一致性修复

### P1
- Step 2 / Step 3 信息层级优化
- Step 4 协作模块强化
- Step 5 导出结果模块强化

### P2
- 案例展示模块
- 可信度内容模块
- 专业版价值模块

## 12. 一句话目标

下一轮重构的目标不是“变好看”，而是让用户更快理解、更敢使用、更愿意把真实研究项目交给这个工具。
