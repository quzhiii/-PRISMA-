# Xiaohongshu Commercial Validation Outreach Guide

Last updated: 2026-06-09

## 1. Goal

Use a human-sounding, low-pressure outreach post to find real systematic-review users who are willing to share workflow pain points and willingness-to-pay signals.

The goal is not to sell, launch pricing, or sound like a formal survey.

## 2. Tone

Recommended tone:

- real builder asking for help
- specific enough to sound credible
- light and readable in a group chat
- not too polished
- no hard CTA

Avoid:

- "market research" language
- long feature dump
- sounding like a startup ad
- asking too many questions at once

## 3. Recommended Group Post

```text
想请教一下群里做过系统综述 / Meta 分析 / 文献筛选的朋友。

我最近在打磨一个本地优先的文献筛选工作台，核心是想把这件事做得更稳一点：导入、去重、筛选、双人复核、质量评价、PRISMA 导出这些都尽量放在一个本地流程里，而且尽量把过程留痕，后面答辩、复核、改稿的时候不至于找不到依据。

最近我在想一个问题：如果这个方向继续往前做，真正值得花钱的点到底在哪里。

不是想卖东西，主要是想少拍脑袋，多听真实做过这类项目的人怎么想。

如果你做过系统综述，想请你随手回我 3-5 句也行：

1. 你现在做文献筛选，最痛的一步是什么？
2. 你更在意「省时间」，还是「过程可复核、能留证据」？
3. 如果有些能力未来可能做成付费增强，你最能接受的是哪类：
   模板包 / AI 辅助 / 双人协作 / 审计报告 / 中文数据库清洗增强 / 机构部署
4. 哪些基础能力你会觉得应该一直免费？
5. 如果你愿意，我也可以私聊发你一个 demo，看你 5 分钟后的真实感受。

如果不方便公开回，也可以私聊我。非常感谢，真的对我判断方向很有帮助。
```

## 4. Shorter Version

```text
想请教一下群里做过系统综述 / Meta 分析的朋友。

我最近在打磨一个本地优先的文献筛选工具，最近最想弄明白的不是功能还能不能继续加，而是：什么东西对真实用户来说是值得付费的，什么东西又必须一直免费。

如果你做过这类项目，想请你随手回我几句：

1. 你做文献筛选最痛的是哪一步？
2. 你更在意省时间，还是更在意过程可复核？
3. 你最可能愿意为哪类增强买单：模板 / AI / 协作 / 审计 / 中文源清洗 / 部署？
4. 哪些基础能力你会觉得必须免费？

公开回或者私聊都行，感谢大家。
```

## 5. Private Follow-Up

Use this when someone replies with interest.

```text
谢谢你愿意回我，真的很有帮助。

我这次主要不是做问卷，更想听真实使用感受，所以不会耽误你太久。

如果方便的话，我想再追问你几个很短的问题：

1. 你之前做系统综述 / 文献筛选，大概是个人做，还是课题组一起做？
2. 你当时最费时间、最容易出错的一步是什么？
3. 你会不会在意以后答辩 / 投稿 / 导师追问时，能不能把筛选过程讲清楚、拿出证据？
4. 如果未来只有一类增强能力值得付费，你直觉最愿意为哪一类付费？为什么？
5. 哪些能力你会觉得一收费就不想用了？

你用最随意的方式回我就行，语音 / 文字都可以。谢谢。
```

## 6. Notes For Use

- Prefer posting at a time when the group is active.
- If the group is warm and informal, use the shorter version.
- If the group is more professional, use the recommended full version.
- Do not post a giant questionnaire first.
- If someone gives a strong reply, move to private follow-up and record it using `docs/commercial/VALIDATION.md`.
